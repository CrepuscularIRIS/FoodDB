# 乳制品供应链风险研判智能体 - 答辩材料 v1.1

## 项目概况

**项目名称**: 乳制品供应链风险研判智能体（知识驱动+规则增强原型）

**核心定位**:
- 不是"预测系统"，而是**"风险研判与监管决策支持系统"**
- 采用**知识驱动+规则增强+模拟数据验证**
- 面向监管动作建议（抽检/溯源/排查）

## 已完成功能（v1.1.0）

### 数据层
- ✅ 6张核心数据表（企业/批次/检验/事件/供应链/GB规则）
- ✅ 模拟数据生成（160批次、30企业、160检验记录）
- ✅ 真实数据集成（上海市监局2025-18通告，6.25%真实数据）

### 规则引擎
- ✅ 9维度风险评分模型
  - 产品类别敏感度
  - 原料来源复杂度
  - 供应商历史风险
  - 批次追溯完整性
  - 标签与标准一致性
  - 历史抽检异常
  - 行政处罚次数
  - 冷链要求敏感度
  - 销售区域扩散度
- ✅ GB标准规则库（GB19301/19645/25190/2760）

### Agent工作流
```
用户输入 → 识别对象 → 取数 → 匹配GB规则 → 计算风险 → 生成报告
```

## 答辩要点

### 1. 为什么用规则引擎而不是GNN？
- **可解释性**: 规则触发可逐条解释，符合监管审计要求
- **可控性**: 权重可调，专家知识可直接注入
- **基础验证**: 先验证知识框架，再逐步替换为学习模型
- **升级路径**: HGNN将在v2.0替换规则引擎的"异构图等效功能"

### 2. 数据真实性如何保证？
- **数据分层**: 6.25%真实数据验证框架可行性
- **来源权威**: 上海市监局公开抽检数据
- **数据冻结**: release_v1_1版本固定，演示可复现

### 3. 与现有系统的区别？
- **知识驱动**: 基于GB标准规则，非黑盒预测
- **供应链视角**: 不止单点检验，关注网络传播
- **决策支持**: 输出抽检/溯源建议，不止风险分数

### 4. 下一步工作？
- **HGNN**: 学习供应链隐式特征
- **MAPPO**: 多智能体协同抽检策略
- **ST-GNN**: 时空风险预测
- **真实数据**: 扩大真实数据比例
