## 八、Code Review 修复记录

### Codex Review 发现的问题及修复

| 问题 | 严重程度 | 状态 | 修复方式 |
|------|----------|------|----------|
| API响应格式不匹配 | Critical | ✅ 已修复 | 解包后端 `{success,data}` 信封 |
| Modal竞态条件 | High | ✅ 已修复 | 添加 `cancelled` flag + 状态重置 |
| URL未编码 | Medium | ✅ 已修复 | 使用 `encodeURIComponent()` |
| 模糊匹配风险 | Medium | ✅ 已修复 | 改为精确名称匹配 |
| 日期跨边界错误 | Medium | ✅ 已修复 | 使用 `startOfDay` 计算 dayDiff |
| localStorage验证 | Low | ✅ 已修复 | 添加 `Array.isArray()` 检查 |

### 修复详情

**1. API响应格式 (frontend/lib/api.ts)**
```typescript
// 修复前: 直接返回 response.data
return { success: true, data: response.data };

// 修复后: 解包后端信封
const payload = response.data;
return {
  success: !!payload?.success,
  data: payload?.data,
  error: payload?.error,
};
```

**2. Modal竞态条件 (EnterpriseModal.tsx / BatchModal.tsx)**
```typescript
// 修复前: 无取消机制
useEffect(() => {
  fetchData(); // 可能覆盖新状态
}, [id]);

// 修复后: 添加取消标志
useEffect(() => {
  let cancelled = false;
  setLoading(true); setError(null); // 重置状态
  fetchData().then(result => {
    if (!cancelled) setData(result);
  });
  return () => { cancelled = true; };
}, [id]);
```

**3. URL编码 (frontend/lib/api.ts)**
```typescript
// 修复前
await api.get(`/enterprises/${id}`);

// 修复后
await api.get(`/enterprises/${encodeURIComponent(id)}`);
```

**4. 日期计算 (frontend/lib/history.ts)**
```typescript
// 修复前: 跨月/年边界错误
if (date.getDate() === now.getDate() - 1) // 3月1日 vs 2月28日

// 修复后: 使用 startOfDay
const dayDiff = Math.floor((startOfDay(now) - startOfDay(date)) / (24*60*60*1000));
if (dayDiff === 1) // 正确计算
```
