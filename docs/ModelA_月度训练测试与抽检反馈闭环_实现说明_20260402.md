# ModelA 月度训练测试与抽检反馈闭环实现说明（2026-04-02）

## 一、这次新增了什么

围绕老师提出的目标（按两文件构建异构图 + 7类风险概率 + 高中低分级 + 抽检反馈闭环），本次在现有 ModelA v2 上新增了“月度训练/测试模拟与抽检反馈”模块：

1. 后端新增接口：`POST /api/modela/v2/temporal_simulate`
2. 前端 ModelA v2 页面新增模块：
   - **月度训练/测试 + 抽检反馈闭环（流程验证）**
   - 可配置训练月、测试月、Top-K、抽检数量、探索权重
   - 可展示反馈前后 Precision@K / Recall@K、抽检命中率、风险分层变化、抽检样本表
3. 对 `screening/ranking_eval/resource_plan` 增加了可调规模参数：
   - `max_nodes`、`max_edges`
   - 并下调默认规模，避免前端接口卡顿

## 二、核心思路（可对外讲）

1. **图构建基线不变**：
   - 节点来自 `enterprise_node.csv`
   - 边来自 `graph_edges_reformatted_with_product.csv`
   - 7类风险概率在节点和边上都输出

2. **月度训练/测试模拟**：
   - 节点保持一致
   - 边按月份进行确定性扰动（同样输入、同样月份、同样seed可复现）
   - 构造“训练月图”和“测试月图”

3. **打分与排序**：
   - 在月度图上继续使用 5.4Pro 公式链路（risk/credibility/uncertainty、piecewise、KQV）
   - 输出 priority_score 作为排序依据

4. **抽检反馈闭环**：
   - 用 `priority + explore(uncertainty)` 选取抽检对象
   - 抽检结果回写（有问题=1，无问题=0）
   - 未抽检样本保持模型/校准分
   - 反馈前后指标对比（Precision@K、Recall@K）

## 四、接口说明（新）

### `POST /api/modela/v2/temporal_simulate`

请求体示例：

```json
{
  "train_month": "2025-01",
  "test_month": "2025-02",
  "product_type": "巴氏杀菌乳",
  "node_type": "原奶供应商",
  "max_nodes": 1200,
  "max_edges": 5000,
  "top_ratio": 0.05,
  "top_k": 20,
  "inspect_count": 40,
  "explore_weight": 0.35,
  "seed": 42
}
```

返回核心字段：

- `train_snapshot` / `test_snapshot`
- `metrics_before` / `metrics_after_feedback`
- `inspection`（抽检命中率、样本列表）
- `risk_buckets_before` / `risk_buckets_after_feedback`
- `recommendations`
